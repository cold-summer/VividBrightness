# Security Policy

## Supported Versions

Security fixes are provided for the latest released version of VividBrightness.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub Private Vulnerability Reporting when it is enabled for the repository. If it is unavailable, contact a maintainer privately through the contact information on their GitHub profile.

Include:

- The affected version or commit
- macOS and hardware details
- Reproduction steps or a proof of concept
- The expected impact
- Any suggested mitigation

Avoid including credentials, personal information, or unrelated system logs. Maintainers should acknowledge a report within seven days and provide status updates as investigation proceeds.
